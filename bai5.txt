1) git init
Tác dụng: Khởi tạo kho lưu trữ 1 git 1 dự án mới hoặc đã có.
Cách sử dụng: git init in the root directory of the project.
2) git thêm
The use: To give a file into Staging Area
Cách sử dụng: git thêm tên_tệp hoặc muốn thêm hết tệp của thư mục, git thêm tất cả
3) bản sao git
Tác dụng: Sao chép kho lưu trữ 1 git từ nguồn từ xa.
Cách sử dụng: git clone <: clone git url:>
4) trạng thái git
Tác dụng: Để kiểm tra trạng thái của những tập tin bạn đã thay đổi trong công việc thư mục. VD: Tất cả các cuối cùng thay đổi từ cuối lần cam kết.
How used: git status in work directory.
5) git thêm
Tác dụng: Thêm thay đổi đến stage / index trong công việc thư mục.
Cách sử dụng: git add
6) git cam kết
Tác dụng: cam kết nghĩa là một hành động để lưu lại ảnh chụp nhanh của các thay đổi trong công việc thư mục. Và các tập tin, thư mục được thay đổi phải nằm trong Khu vực dàn dựng. Mỗi lần cam kết nó sẽ được lưu lại sửa chữa lịch sử sửa mã kèm theo tên và địa chỉ email của người cam kết. Ngoài ra trong Git, bạn cũng có thể khôi phục tập tin trong lịch sử cam kết của nó để chia cho một chi nhánh khác, vì vậy bạn sẽ dễ dàng khôi phục lại các thay đổi trước đó.
Cách sử dụng: git commit -m ”This is message, you used to note the changes to after this view”
7) git push / git pull
Tác dụng: Đẩy hoặc kéo các thay đổi đến từ xa. Nếu bạn đã thêm và cam kết các thay đổi và bạn muốn đưa nó lên hoặc từ xa của bạn đã cập nhật và bạn áp dụng tất cả các thay đổi đó trên mã của mình.
Cách sử dụng: git pull <: remote:> <: branch:> và git push <: remote:> <: branch:>
